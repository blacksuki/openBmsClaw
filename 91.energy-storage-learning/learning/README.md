# Learning

本目录保存学习过程材料。

学习材料允许逐步完善，但必须做到：

1. 标题清楚。
2. 日期清楚。
3. 结论、证据、待验证项分开。
4. 能回链到对应的 roadmap day。

## 子目录

| 目录 | 用途 |
|---|---|
| [notes/](notes/README.md) | 每日学习笔记、概念卡片、访谈记录 |
| [diagrams/](diagrams/README.md) | 市场图、供应链图、系统结构图 |
| [summaries/](summaries/README.md) | 阶段总结、主题摘要、一页纸 |
| [homework/](homework/README.md) | 作业、测验、导师批改稿 |

## Day 文件归档规则

1. `notes/`
   - 放 Day 学习笔记、技术卡片、导师讲解稿、学习计划
   - 例如：`Day02_japan-compliance-route-v0.md`
   - 例如：`Day03_low-voltage-storage-core-knowledge-cards.md`
2. `homework/`
   - 放 Day 作业入口索引、作业过程稿、reviewed/revision 稿
   - 不与 `research/` 重复保存最终归档版
3. 如果一份 Day 作业已经成为“证据研究文档”，应归档到 `research/`，并在 `homework/README.md` 里留索引
4. 如果一份 Day 输出已经直接服务业务判断，应归档到 `project/`

## 文件命名建议

```text
Day01_japan-low-voltage-storage-scenarios.md
Day02_japan-compliance-route.md
Day03_core-technology-cards.md
Week01_review.md
```

保留 `DayXX` 前缀，便于和 [../roadmap/](../roadmap/README.md) 对齐。
