## 4. 今日学习方式

今天按“项目决策顺序”学习，不按法规章节顺序学习。

### 阶段 A：场景回看

目标：
只提炼两条候选样板线的真实约束。

必须回答：

1. 公寓防灾用户真正要解决什么问题？
2. 办公移动电源用户真正要解决什么问题？
3. 两者分别更像“灾害待机产品”还是“高频日常使用产品”？

### 阶段 B：产品分型

目标：
把产品先分型，再谈约束。

必须回答：

1. 公寓防灾样板线更像 `AC 便携储能`、`DC 外接电源` 还是混合形态？
2. 办公移动电源样板线更像 `个人随身插座`、`共享补电节点` 还是传统户外电源？
3. 两条线是否默认带无线、App、BLE 或云端管理？

### 阶段 C：合规映射

目标：
为每条样板线建立 `主约束 -> 次约束 -> 资料包`。

至少要映射的体系：

1. `PSE / 电安法`
2. `JET / JIS / S-JET`
3. `TELEC`
4. `JC-STAR`
5. `UN38.3 / SDS / 运输资料`

### 阶段 D：样板线比较

目标：
比较哪一条线更适合作为第一条样板线。

比较维度：

1. 产品边界是否容易讲清
2. 合规边界是否容易解释
3. 资料包是否容易先做出来
4. 是否更贴近 `openBattery` 的差异化能力
5. 风险是否更可控

## 6. 今日导师监督要求

今天不接受以下三类输出：

1. 只有法规名词，没有产品分型。
2. 只有认证名字，没有资料包和适用边界。
3. 不给出样板线判断，只停留在“都要研究”。

今天必须显式区分：

1. `官方可确认`
2. `合理推论`
3. `待验证`
4. `需询价`
