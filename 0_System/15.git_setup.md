# Git 初始化与仓库设置

> 最新核对日期：2026-06-25

## 1. 文档目的
本文档用于给 `openBattery` / `openBmsClaw` 仓库提供一份适合当前项目状态的 Git 初始化建议。

当前前提：

- Git root：仓库根目录（`openBmsClaw/` 工程所在的 `openBattery/` 仓库根，以下命令均在此目录执行）
- 已执行：`git init`
- 尚未执行：首次 `add / commit / branch` 整理

当前仓库特点：

- 既有系统文档、计划文档
- 有 `openBmsClaw/` 这样的 STM32 工程
- 有 `90_documents/`、`91.reference/` 这样的参考资料目录
- 同时存在 CMake 生成物、STM32 IDE 状态文件和 macOS 本地垃圾文件

因此 Git 设置重点不是“复杂工作流”，而是先把：

1. 忽略规则
2. 行尾规则
3. 仓库本地配置
4. 第一次提交顺序

固定下来。

## 2. 当前建议保留在版本控制中的内容

建议入库的内容：

- `0_System/`
- `1_Plan/`

- `AGENTS.md`

- `.DS_Store`

## 3. 建议的仓库本地 Git 设置

建议在仓库根目录执行下面这些命令：

```bash
git branch -m main
git config --local core.autocrlf input
git config --local core.quotepath false
git config --local fetch.prune true
git config --local pull.ff only
```

这些设置的作用分别是：

- `branch -m main`
  - 把初始分支改成 `main`
  - 当前还没有提交，改名成本最低

- `core.autocrlf input`
  - 在 macOS 上保留本地 `LF`
  - 避免把 Windows 风格行尾不必要地写进仓库

- `core.quotepath false`
  - `git status` 显示中文路径时不再是一串转义字符
  - 对当前这个中文资料较多的仓库非常重要

- `fetch.prune true`
  - 拉取远端时自动清理已删除分支的远端引用

- `pull.ff only`
  - 默认只允许快进合并
  - 先避免新手阶段无意中制造多余 merge commit

## 4. 用户身份设置

如果当前机器还没有配置 Git 用户身份，第一次提交前至少要设置：

```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"
```

如果你不想影响全局环境，也可以只给当前仓库设置：

```bash
git config --local user.name "你的名字"
git config --local user.email "你的邮箱"
```

## 5. 第一次提交前的建议检查

建议按这个顺序检查：

```bash
git status --short
git add .gitignore .gitattributes AGENTS.md
git add 0_System 1_Plan 2_Action openBmsClaw
git status
```

在 `status` 里重点确认：

- `build/` 没有被加进去
- `.settings/` 没有被加进去
- 根目录 `.vscode/settings.json` 没有被加进去
- 当前主线文档和 `openBmsClaw/` 代码都已在 staged 列表中

建议：

- 第一笔提交先只固定当前主线：`AGENTS.md`、`0_System/`、`1_Plan/`、`2_Action/`、`openBmsClaw/`
- `90_documents/` 和 `91.reference/` 建议作为第二笔“参考资料归档”提交，先人工确认再加入

---

## 6. 第一次提交建议

如果上面检查没问题，第一笔提交建议使用：

```bash
git commit -m "chore: initialize openBattery repository baseline"
```

这笔提交的定位应是：

- 建立仓库基础结构
- 固定文档、计划和当前 STM32 工程
- 固定忽略规则和 Git 规范

不要把它写成“功能完成”或“平台已跑通”。

## 7. 当前仓库的额外提醒

### 7.1 根目录 `.vscode/`
当前根目录 `.vscode/settings.json` 使用了绝对路径：

- `cmake.sourceDirectory: /Users/huoward/.../openBmsClaw`

这类文件不适合直接共享，因此仓库级 `.gitignore` 已忽略根目录 `.vscode/`。

### 7.2 子工程 `.settings/`
`STM32CubeIDE for Visual Studio Code` 生成的 `.settings/*.store.json` 更接近本地 IDE 状态，不建议纳入版本控制。

### 7.3 参考资料目录
`90_documents/` 和 `91.reference/` 当前承担参考和归档角色，不要因为它们里有二进制文件，就一刀切全部忽略。

但也不要在第一次提交时直接无脑全加。

更合理的方式是：

- 第一笔提交先固定当前主线
- 第二笔提交再筛选要归档的参考资料
- 本地解压出来的 `.app` 安装包目录默认不纳入版本控制
