## 1. 文档目的

本文档用于给 `openBattery` 项目提供一份当前可执行的采购清单，并统一下面几个口径：

- 当前学习和开发硬件是什么。
- 未来生产环境预计使用的 MCU 候补路线是什么。
- 哪些器材能直接服务 `F103 bring-up -> F030 量产候补验证 -> 专用 SoC 协同` 主线。
- 哪些器材只是回退、扩展或后续阶段再考虑。

本文档遵循当前主线：

- 当前学习 / bring-up 硬件：`野火小智 STM32F103C8T6 核心板（双 USB 款）`
- 未来量产候补路线：`F030` 级别 MCU，优先评估 64KB Flash 档位样片
- 当前工程目标：先完成最小可运行验证，再逐步接入英集芯、智融等专用电源 SoC

---

## 2. 采购策略结论

### 2.1 主控策略

- 当前学习/起步平台：`野火小智 STM32F103C8T6 核心板（双 USB 款）` + 外部 `SWD` 调试器
- 量产候补样片：`STM32F030C8T6`、`HK32F030C8T6/R8T6`、`CKS32F030C8T6/R8T6`
- 不建议作为默认量产候补：16KB Flash / 2KB SRAM 级别的 `F030M` 小资源器件
- 回退平台：若 F030 样片验证失败，再评估 `STM32G0`、`GD32E230` 或其它资源更宽裕 MCU

这里要固定两个概念：

- `F103` 是当前学习和开发入口，不是量产定板结论。
- `F030` 是生产环境预计使用的量产候补，必须通过样片、资源、外设和 SoC 协同验证后才能写入量产参考设计。

### 2.2 面向价格敏感客户的建议

| 档位 | 建议方案 | 适用场景 |
| --- | --- | --- |
| 教学/验证极简版 | `STM32F103C8T6` 学习板 + 外部 `SWD` 调试器 | 下载、调试、GPIO / ADC / I2C / UART 入门验证 |
| 量产候补验证版 | `STM32F030C8T6` / `HK32F030C8T6/R8T6` / `CKS32F030C8T6/R8T6` 样片或最小板 | 验证 F030 是否满足未来生产环境最低要求 |
| 资源回退版 | `STM32G0` / `GD32E230` / 其它更宽裕 MCU | F030 无法满足资源、调试或可靠性要求时再启用 |

原则是：

- 当前开发不要脱离 F103 学习板和现有工程事实。
- 量产降本不要只看单价，必须同时验证 Flash/SRAM、I2C、ADC、EXTI、watchdog 和调试稳定性。
- F030 验证失败时，才把更宽裕 MCU 作为回退主线，而不是提前把 G0 写成默认路线。

## 3. 主控与调试采购表

| 类别 | 产品名称 | 推荐级别 | 用途 | 当前建议 |
| --- | --- | --- | --- | --- |
| 学习/起步板 | `野火小智 STM32F103C8T6 核心板（双 USB 款）` | 已有/可用 | 当前学习与最小验证 | 继续使用 |
| 调试器 | `ST-LINK/V2` | 已有/可复用 | 现阶段下载与调试 | 可继续复用 |
| 调试器 | `STLINK-V3MINIE` | 建议 | 后续统一标准调试器 | 下批采购补齐 |
| 量产候补样片 | `STM32F030C8T6` | 必买 | F030 资源与工具链基准 | 用作基准对照 |
| 量产候补样片 | `HK32F030C8T6/R8T6` | 建议 | 成本候补 A | 验证封装、资源、SWD、I2C、ADC |
| 量产候补样片 | `CKS32F030C8T6/R8T6` | 建议 | 成本候补 B | 验证 HAL/寄存器兼容性和 SRAM 余量 |
| 回退开发板 | `STM32G0` 或 `GD32E230` 参考板 | 暂缓 | F030 验证失败后的资源回退 | 不作为当前默认主线 |

补充说明：

- `ST-LINK/V2` 当前已足够支撑 `SWD` 下载、断点调试和 `CubeProgrammer` 烧录。
- `STLINK-V3MINIE` 更适合后续自研板和量产候补板的长期调试。
- 采购 F030 样片时，优先选择 64KB Flash 档位；不要用 16KB 小资源型号代表整条 F030 路线。

## 4. 专用 SoC 与电源路径采购表

当前架构是“专用 SoC 负责体力活，MCU 负责脑力活”，所以应优先采购能暴露 I2C/UART/INT 接口的电源 SoC 评估板。

| 产品名称 | 推荐级别 | 适用目标 | 备注 |
| --- | --- | --- | --- |
| 英集芯 IP53xx 系列 DEMO / EVB | 必买 | 首条 SoC 协同链路 | 重点确认寄存器手册、I2C 地址、INT 告警脚 |
| 智融 SW35xx 系列 DEMO / EVB | 建议 | 第二 SoC 适配对象 | 用于验证 `drivers/` SAL 是否真正可替换 |
| `BQ25792EVM` | 后续建议 | 更贴近移动电源充放电链路 | 进入独立 power 模块时再采购 |
| `BQ76942EVM` | 后续建议 | 官方完整 BMS 评估系统 | 多串 BMS 深入验证时再采购 |
| `EV2400` | 搭配件 | 与 TI 部分评估板通信 | 视 TI EVM 使用方式决定 |

## 5. I2C / 学习型扩展件采购表

这一批器材价格低、国内购买方便，而且和当前开源框架直接相关：

| 产品名称 | 推荐级别 | 作用 |
| --- | --- | --- |
| `0.96 寸 I2C OLED` | 必买 | 推进 `I2C` 驱动、状态显示和调试可视化 |
| `INA219` 模块 | 必买 | 推进电流、电压、功率测量接口 |
| `24LC02B / AT24C02` 模块 | 必买 | 推进 `I2C` 地址扫描、读写寄存器、配置存储 |
| `NTC` 热敏电阻和分压电阻 | 必买 | 验证 ADC 温度采样、滤波和热保护策略 |
| `BMP280` 模块 | 可选 | 继续熟悉 `I2C` 传感器读写 |

这类模块的价值在于：

- 能帮助定义 `board/hal/drivers/services` 的边界。
- 能在接入真实 SoC 前先验证 I2C 超时、错误码和日志。
- 比直接上真实 BMS 板更适合新手快速建立驱动经验。

## 7. 当前推荐组合

### 7.1 最适合现在的组合

- 当前已有：`STM32F103C8T6` 学习板 + `ST-LINK/V2`
- 新增采购：
  - `STM32F030C8T6` 样片或最小板
  - `HK32F030C8T6/R8T6` 或 `CKS32F030C8T6/R8T6` 至少一种国产候补样片
  - 英集芯 IP53xx 或智融 SW35xx DEMO / EVB
  - `INA219`
  - `I2C OLED`
  - `24LC02B / AT24C02`
  - `NTC` 与分压电阻
  - 逻辑分析仪

这套组合最符合当前项目目标：

- 继续当前 F103 学习节奏。
- 尽快进入 F030 量产候补样片验证。
- 同时积累 `I2C / ADC / SoC SAL / thermal` 的可复用驱动和测试文档。

### 7.2 不建议现在的组合

- 只继续买 `F103` 周边，不准备 F030 样片。
- 一上来就重押昂贵 `BMS EVM + 电池包 + 多串测试夹具`。
- 在 F030 未验证前，把 `STM32G0` 或其它更大 MCU 写成默认量产主线。
- 为了省成本选择 16KB Flash 小资源 F030M 作为平台默认器件。

这些做法都会让当前开源项目偏离“F103 学习开发 -> F030 量产候补验证 -> SoC 协同最小链路”的主干线。
