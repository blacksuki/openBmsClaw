# 分层采购建议

> 最新核对日期：2026-06-25

## 1. 文档目的

本文档用于把当前 `openBattery` 项目的采购建议收敛成一份更适合执行的分层清单，重点解决下面几个问题：

- 哪些器材现在就该买。
- 哪些器材建议买，但不应阻塞当前开发。
- 哪些器材应暂缓，等 F030 量产候补主线验证后再投入。
- 预算有限时，应该优先买哪些最能推进当前主干线的器材。

本文档遵循当前项目主线：

- 当前学习/起步硬件：`野火小智 STM32F103C8T6 核心板（双 USB 款）`
- 未来量产候补 MCU：`F030` 级别 64KB Flash 档位器件
- 当前开源目标：构建一套可定制、低成本、可量产迁移的模块化移动电源 / BMS 平台

这意味着采购不应只服务“单片机入门”，而应优先服务：

- F103 当前学习和开发。
- F030 样片、最小板和资源预算验证。
- `I2C / UART / ADC / EXTI / watchdog` 等可复用驱动能力。
- 英集芯、智融等专用 SoC 的协同链路。

---

## 2. 采购结论

### 2.1 必买

这批器材最符合当前项目主线，也最能兼顾“学习”和“量产候补验证”：

| 类别 | 产品名称 | 当前建议 | 作用 |
| --- | --- | --- | --- |
| 调试器 | `ST-LINK/V2` | 已有则继续复用 | 当前 F103 学习板和后续 F030 候补板的 `SWD` 下载和调试 |
| MCU 样片 | `STM32F030C8T6` | 必买 | 作为 F030 生态、资源和工具链基准 |
| MCU 样片 | `HK32F030C8T6/R8T6` 或 `CKS32F030C8T6/R8T6` | 至少买一种 | 验证国产 F030 候补是否适合量产降本 |
| SoC 评估板 | 英集芯 IP53xx 或智融 SW35xx DEMO / EVB | 必买 | 打通 MCU 与专用电源 SoC 的 I2C/UART/INT 协同链路 |
| I2C 学习件 | `0.96 寸 I2C OLED` | 必买 | 推进 `I2C` 驱动、状态显示和板级联调 |
| I2C 学习件 | `INA219` 模块 | 必买 | 推进电流、电压、功率测量接口 |
| I2C 学习件 | `24LC02B / AT24C02` 模块 | 必买 | 推进 `I2C` 地址扫描、读写寄存器、配置存储 |
| ADC 学习件 | `NTC` 与分压电阻 | 必买 | 验证温度采样、滤波和热保护策略 |
| 仪器 | 逻辑分析仪 | 强烈推荐 | 观测 I2C 总线死锁、Clock Stretching 和恢复机制 |

### 2.2 建议

这批器材能明显提高推进效率，但不必要求第一天全部到位：

| 类别 | 产品名称 | 当前建议 | 作用 |
| --- | --- | --- | --- |
| 调试器 | `STLINK-V3MINIE` | 建议补齐 | 作为长期标准调试器，更适合后续自研板和量产候补板 |
| 仪器 | 电子负载 | 建议 | 测试高负载温升与功率限制调度 |
| 仪器 | 示波器 | 建议 | 观察纹波、电源噪声和信号边沿 |
| 电源路径板 | `BQ25792EVM` | 后续建议 | 更贴近移动电源充放电链路，适合进入独立 power 模块阶段 |
| 电池评估板 | `BQ76942EVM` | 后续建议 | 适合后续真实 `BMS` 驱动接入和多串保护验证 |

### 2.3 暂缓

这批器材不是没价值，而是当前阶段容易投入过早：

| 类别 | 产品名称 | 当前建议 | 原因 |
| --- | --- | --- | --- |
| 回退开发板 | `STM32G0` / `GD32E230` 等更宽裕 MCU 开发板 | 暂缓 | 作为 F030 验证失败后的回退路线，不作为当前默认主线 |
| 小资源 MCU | 16KB Flash / 2KB SRAM 级别 F030M | 暂缓 | 不适合承载当前平台最低可观测和分层目标 |
| 高阶主控板 | `STM32F4 / WB` 高配板 | 暂缓 | 当前主线还没把 `F030 + SoC + I2C + ADC` 跑实 |
| 多串 BMS 深水区器材 | 电池包夹具、大功率电子负载、大量 pack 配套件 | 暂缓 | 在 `BMS / power` 软件结构还未稳定前，投入回报偏低 |

## 3. 为什么这套采购更贴近当前主干线

当前项目不是做单片机练习题，而是要把下面这些能力真正落进开源框架：

- `board`
- `hal`
- `drivers` / SoC SAL
- `services`
- `app_min`
- `config`

因此采购要优先支持：

1. 当前 F103 学习板能稳定下载、调试和输出日志。
2. F030 候补器件能完成样片级外设验证。
3. 专用 SoC 能提供清晰的 I2C/UART/INT 接口。
4. 低成本 I2C/ADC 模块能先验证驱动边界。
5. 逻辑分析仪能帮助定位 I2C 死锁和恢复问题。

## 4. 预算档位建议

### 4.1 低预算档

适合当前以学习和平台骨架推进为主，不急于马上进入真实电池评估板：

| 采购项 | 是否建议 |
| --- | --- |
| 继续使用现有 `ST-LINK/V2` | 是 |
| `STM32F030C8T6` 样片或最小板 | 是 |
| `HK32F030C8T6/R8T6` 或 `CKS32F030C8T6/R8T6` 至少一种 | 是 |
| `0.96 寸 I2C OLED` | 是 |
| `INA219` 模块 | 是 |
| `24LC02B / AT24C02` 模块 | 是 |
| `NTC` 与分压电阻 | 是 |

特点：

- 预算相对克制。
- 能验证 F030 是否有机会进入生产环境。
- 能完成 `F030 + I2C + ADC + 基础日志` 的第一轮实质性推进。

### 4.2 中预算档

适合希望更快进入 SoC 协同链路：

在低预算档基础上增加：

- `STLINK-V3MINIE`
- 英集芯 IP53xx 或智融 SW35xx DEMO / EVB
- 逻辑分析仪
- USB 功率表

特点：

- 调试器标准化。
- 能开始验证 MCU 与专用 SoC 的真实通信。
- 更适合沉淀 F030 bring-up 和 SoC SAL 文档。

### 4.3 高预算档

适合准备进入真实 `power + BMS` 模块接入：

在中预算档基础上增加：

- 电子负载
- 示波器
- `BQ25792EVM`
- `BQ76942EVM`

特点：

- 可以开始做真实芯片级模块接入。
- 更接近后续开源框架里 `power` 和 `bms` 的真实抽象。
- 但对新手来说，文档阅读和联调难度会明显上升。

## 5. 当前推荐组合

### 5.1 最适合现在的组合

- 当前已有：`F103` 学习板 + `ST-LINK/V2`
- 新增采购：
  - `STM32F030C8T6`
  - `HK32F030C8T6/R8T6` 或 `CKS32F030C8T6/R8T6`
  - 英集芯 IP53xx 或智融 SW35xx DEMO / EVB
  - `INA219`
  - `I2C OLED`
  - `24LC02B / AT24C02`
  - `NTC` 与分压电阻
  - 逻辑分析仪

这套组合最符合当前项目目标：

- 既能继续当前 F103 学习节奏。
- 又能验证 F030 是否符合未来生产环境最低要求。
- 同时开始积累 `I2C / ADC / SoC SAL / thermal` 的可复用驱动和测试文档。

### 5.2 不建议现在的组合

- 只继续买 `F103` 周边。
- 不买 F030 样片，直接讨论量产定板。
- 一上来就重押昂贵 `BMS` 评估板。
- 在 F030 样片未验证前，把更大 MCU 开发板写成默认主线。

这些做法都会让当前开源项目偏离“先最小可运行，再围绕 F030 量产候补逐步平台化”的路线。
