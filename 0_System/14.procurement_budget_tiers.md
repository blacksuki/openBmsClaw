# 分层采购建议

> 最新核对日期：2026-04-08

## 1. 文档目的
本文档用于把当前 `bmsClaw` 项目的采购建议收敛成一份更适合执行的分层清单，重点解决下面几个问题：

- 哪些器材现在就该买
- 哪些器材建议买，但不应阻塞当前开发
- 哪些器材应暂缓，等平台主线稳定后再投入
- 预算有限时，应该优先买哪些最能推进开源平台的器材

本文档遵循当前项目主线：

- 长期平台 MCU 路线：`STM32G0`
- 当前学习/起步硬件：`野火小智 STM32F103C8T6 核心板（双 USB 款）`
- 当前开源目标：构建一套可定制、低成本、可量产迁移的模块化移动电源 / BMS 平台

这意味着采购不应只服务“单片机入门”，而应优先服务：

- `STM32G0` 参考主线
- `USB-C / PD` 输入与双角色链路
- `I2C / ADC / 电流电压测量` 等可复用驱动能力
- 后续 `power / protocol / bms` 三类模块接入

---

## 2. 采购结论

### 2.1 必买
这批器材最符合当前项目主线，也最能兼顾“学习”和“开源平台推进”：

| 类别 | 产品名称 | 当前建议 | 作用 |
| --- | --- | --- | --- |
| 调试器 | `ST-LINK/V2` | 已有则继续复用 | 当前 `F103` 学习板与后续 `STM32G0` 板都可用于 `SWD` 下载和调试 |
| 参考开发板 | `NUCLEO-G071RB` | 必买 | `STM32G0` 默认参考开发板，适合作为长期主线起点 |
| 协议扩展板 | `X-NUCLEO-DRP1M1` | 必买 | 第一块推荐 `USB Type-C / PD` 板，适合做输入链路和 `DRP` 角色切换 |
| I2C 学习件 | `0.96 寸 I2C OLED` | 必买 | 低成本推进 `I2C` 驱动、UI 状态显示和板级联调 |
| I2C 学习件 | `INA219` 模块 | 必买 | 推进电流、电压、功率测量接口，贴近后续电源路径观测 |
| I2C 学习件 | `24LC02B / AT24C02` 模块 | 必买 | 推进 `I2C` 地址扫描、读写寄存器、配置存储 |

### 2.2 建议
这批器材能明显提高推进效率，但不必要求第一天全部到位：

| 类别 | 产品名称 | 当前建议 | 作用 |
| --- | --- | --- | --- |
| 调试器 | `STLINK-V3MINIE` | 建议补齐 | 作为长期标准调试器，更适合后续自研板和 `STM32G0` 主线 |
| 参考开发板 | `NUCLEO-G0B1RE` | 建议 | 资源更宽裕，适合协议、显示和多模块并行验证 |
| 一体化 PD 板 | `STM32G071B-DISCO` | 建议 | 想更快熟悉 `USB Type-C / PD` 参考设计时很有价值 |
| 电源路径板 | `BQ25792EVM` | 建议 | 更贴近移动电源充放电链路，适合进入 `power` 模块阶段 |
| 电池评估板 | `BQ76942EVM` | 建议 | 适合后续真实 `BMS` 驱动接入和多串保护验证 |

### 2.3 暂缓
这批器材不是没价值，而是当前阶段容易投入过早：

| 类别 | 产品名称 | 当前建议 | 原因 |
| --- | --- | --- | --- |
| 量产 MCU 样片 | `STM32G030C8T6` 等低成本 MCU | 暂缓 | 适合量产裁剪评估，不适合替代当前参考开发板 |
| 高阶主控板 | `STM32F4 / WB` 高配板 | 暂缓 | 当前主线还没把 `G0 + PD + I2C + 基础 power` 跑实 |
| 多串 BMS 深水区器材 | 电池包夹具、大功率电子负载、大量 pack 配套件 | 暂缓 | 在 `BMS / power` 软件结构还未稳定前，投入回报偏低 |

## 3. 为什么这套采购更贴近 open 项目

### 3.1 它不是“只买学习板”
当前项目不是做单片机练习题，而是要把下面这些能力真正落进开源框架：

- `board`
- `drivers`
- `services/power`
- `services/protocol`
- `services/bms`

因此采购要优先支持：

- 能做 `SWD` 下载与调试
- 能做 `I2C / ADC / UART` 基础驱动验证
- 能做 `USB-C / PD` 协议链路验证
- 能做电压 / 电流 / 功率的最小观测

### 3.2 它不是“现在就重押 BMS”
当前最容易把新手拖死的路径，是一上来就投入昂贵 `BMS EVM + 电池包 + 多串保护`，但主控侧基础驱动和协议链路还没稳定。

更合理的顺序是：

1. `F103` 学习板继续完成基础 bring-up 和记日志
2. 转入 `STM32G0` 参考开发板
3. 接入第一块 `PD` 板
4. 补齐 `INA219 / OLED / EEPROM` 这类低成本 I2C 模块
5. 再进入真实 `power / BMS` 评估板

这个顺序更符合当前项目“先最小可运行，再逐步平台化”的路线。

## 4. 预算档位建议

### 4.1 低预算档
适合当前以学习和平台骨架推进为主，不急于马上进入真实电池评估板：

| 采购项 | 是否建议 |
| --- | --- |
| 继续使用现有 `ST-LINK/V2` | 是 |
| `NUCLEO-G071RB` | 是 |
| `X-NUCLEO-DRP1M1` | 是 |
| `0.96 寸 I2C OLED` | 是 |
| `INA219` 模块 | 是 |
| `24LC02B / AT24C02` 模块 | 是 |

特点：

- 最能推进当前开源主线
- 预算相对克制
- 能完成 `STM32G0 + I2C + PD` 的第一轮实质性推进

### 4.2 中预算档
适合希望更快进入 `USB-C / PD` 和更完整板级体验：

在低预算档基础上增加：

- `STLINK-V3MINIE`
- `NUCLEO-G0B1RE`
- `STM32G071B-DISCO`

特点：

- 调试器标准化
- `PD` 参考工程更完整
- 更适合后续沉淀统一 bring-up 文档

### 4.3 高预算档
适合准备进入真实 `power + BMS` 模块接入：

在中预算档基础上增加：

- `BQ25792EVM`
- `BQ76942EVM`

特点：

- 可以开始做真实芯片级模块接入
- 更接近后续开源框架里 `power` 和 `bms` 的真实抽象
- 但对新手来说，文档阅读和联调难度会明显上升

## 5. `ST-LINK/V2` 和 `STLINK-V3MINIE` 怎么看

### 5.1 当前结论
如果你手头已经有 `ST-LINK/V2`，**可以继续复用**，不需要因为当前阶段就停下开发去等 `STLINK-V3MINIE`。

更准确的口径是：

- `ST-LINK/V2`：当前可继续用
- `STLINK-V3MINIE`：建议后续补齐，作为长期标准调试器

### 5.2 为什么 `ST-LINK/V2` 还能继续用
对当前你正在做的事情来说，`ST-LINK/V2` 已经覆盖核心需求：

- `SWD` 下载
- `SWD` 断点调试
- `STM32CubeProgrammer` 烧录
- `VS Code / STM32CubeIDE` 联机调试

这足以支撑：

- `F103` 学习板 bring-up
- `NUCLEO-G071RB` 后续主线工程
- 早期 `I2C / ADC / UART / LED` 板级验证

### 5.3 为什么仍推荐 `STLINK-V3MINIE`
`STLINK-V3MINIE` 更适合作为后续统一标准，原因是：

- 更小，更适合长期带着做外部板卡调试
- 调试接口形态更现代
- 支持 `Virtual COM port`
- `SWD / SWV` 速率能力更高

对当前项目来说，它的价值主要在“后续效率和标准化”，不是“当前没有它就不能开发”。

### 5.4 实施建议
建议固定为：

- 现在：继续用现有 `ST-LINK/V2`
- 下一个采购批次：补 `STLINK-V3MINIE`
- 文档口径：两者都能用，但长期推荐 `STLINK-V3MINIE`

## 6. 具体采购顺序

建议按下面顺序下单：

1. `NUCLEO-G071RB`
2. `X-NUCLEO-DRP1M1`
3. `0.96 寸 I2C OLED`
4. `INA219` 模块
5. `24LC02B / AT24C02` 模块
6. `STLINK-V3MINIE`
7. `STM32G071B-DISCO`
8. `BQ25792EVM`
9. `BQ76942EVM`

如果当前预算只能打一枪，优先级应压缩为：

1. `NUCLEO-G071RB`
2. `X-NUCLEO-DRP1M1`
3. `INA219` 模块
4. `I2C OLED`

---

## 7. 当前推荐组合

### 7.1 最适合现在的组合
- 当前已有：`F103` 学习板 + `ST-LINK/V2`
- 新增采购：
  - `NUCLEO-G071RB`
  - `X-NUCLEO-DRP1M1`
  - `INA219`
  - `I2C OLED`
  - `24LC02B / AT24C02`

这套组合最符合当前项目目标：

- 既能继续当前学习节奏
- 又能把开发主线拉回 `STM32G0`
- 同时开始积累 `I2C / PD / power observation` 的可复用驱动和测试文档

### 7.2 不建议现在的组合
- 只继续买 `F103` 周边
- 一上来就重押昂贵 `BMS` 评估板
- 为了省预算直接跳过 `STM32G0` 参考开发板

这些做法都会让当前开源项目更像“零散实验”，而不是“逐步成型的平台”。
