## 1. 文档目的
本文档用于给 `bmsClaw` 项目提供一份当前可执行的采购清单，并统一下面几个口径：

- 长期平台 MCU 路线是什么
- 当前学习硬件和长期参考开发板如何区分
- 哪些协议板、BMS 板和调试器更适合当前阶段
- 采购建议如何与“可定制、低成本、可量产迁移”的目标保持一致

本文档遵循当前主线：

- 长期平台参考路线：`STM32G0`
- 当前学习 / bring-up 硬件：`野火小智 STM32F103C8T6 核心板（双 USB 款）`
- 当前工程目标：先完成最小可运行验证，再逐步接入 `protocol / power / bms`

## 2. 采购策略结论

### 2.1 主控策略
- 当前学习/起步平台：`野火小智 STM32F103C8T6 核心板（双 USB 款）` + 外部 `SWD` 调试器
- 默认参考开发板：`NUCLEO-G071RB`
- 资源更宽裕的参考开发板：`NUCLEO-G0B1RE`
- 长期量产 MCU 路线：`STM32G0` 系列
- 不建议把 `STM32F103C8T6` 写成当前项目的长期主平台

这里要固定两个概念：

- `NUCLEO-G071RB`、`NUCLEO-G0B1RE` 是开发板型号
- `STM32G030C8T6`、`STM32G071RBT6`、`STM32G0B1RET6` 才是量产 MCU 型号

也就是说：

- 开发阶段采购的是参考开发板
- 量产阶段采购的是具体 MCU

### 2.2 方案提供商的推荐方式
作为方案提供商，建议采用下面的表达方式：

- 推荐主控 MCU 家族与参考型号
- 指定必须满足的接口能力、保护边界和验证要求
- 允许在同一软件边界下做量产级成本裁剪

因此更准确的口径应是：

- 推荐：`STM32G0` 作为主平台 MCU 家族，`NUCLEO-G071RB` 作为默认参考开发板
- 指定：接口能力、保护策略、抽象层边界和验证方法
- 允许裁剪：在不破坏接口与验证边界的前提下，为价格敏感客户切换到更低成本的 `STM32G0` 器件

### 2.3 面向价格敏感客户的建议

| 档位 | 建议方案 | 适用场景 |
| --- | --- | --- |
| 教学/验证极简版 | `STM32F103C8T6` 学习板 + 外部 `SWD` 调试器 | 只做下载、调试、GPIO / ADC / I2C / UART 入门验证 |
| 推荐平台预研版 | `NUCLEO-G071RB` / `NUCLEO-G0B1RE` | 当前项目的长期参考开发路线 |
| 成本优化量产版 | 在 `STM32G0` 家族内裁剪，如评估 `STM32G030C8T6` | 接口与软件框架稳定后再做 |

原则是：

- 开发平台不要为了省几美元而偏离主架构
- 真正的成本优化应发生在量产器件和外围选型阶段
- 原型阶段优先降低架构风险，而不是优先压低开发板成本

### 4.2 采购建议

#### 路线 A：先进入真实电源路径
- `BQ25792EVM`

适合：

- 先从充放电、电源路径和 `I2C` 控制切入

- `BQ76942EVM`

- 开始做真实 `BMS` 驱动接入
- 进入多串监测、保护和告警路径

- 分压电阻网络
- 电流采样模块
- `NTC` 热敏电阻
- 可调电源

该路线适合“先验证软件框架”，但不能替代真正的 `BMS` 验证。

这一批器材价格低、国内购买方便，而且和当前开源框架直接相关：

| 产品名称 | 推荐级别 | 作用 |
| --- | --- | --- |
| `0.96 寸 I2C OLED` | 必买 | 推进 `I2C` 驱动、状态显示和调试可视化 |

## 7. ST-LINK 说明

### 7.1 ST-LINK 是什么
`ST-LINK` 是 ST 提供的在线调试器 / 烧录器，用于把固件下载到 `STM32`，并进行联机调试。

### 7.2 它在本方案中的角色
它不是“产品功能模块”，而是开发工具链的一部分。

也就是说：

- 它不是最终客户产品必须具备的功能
- 但它几乎是 `STM32` 开发阶段的标准工具

### 7.3 `ST-LINK/V2` 是否还能继续用
可以。

对当前阶段而言，`ST-LINK/V2` 已足够覆盖：

- `SWD` 下载
- `SWD` 断点调试
- `STM32CubeProgrammer` 烧录
- `VS Code / STM32CubeIDE` 联机调试

因此：

- 当前手头已有 `ST-LINK/V2` 时，不必停下开发去等 `STLINK-V3MINIE`
- 但长期标准化仍建议补齐 `STLINK-V3MINIE`

### 7.4 为什么仍推荐 `STLINK-V3MINIE`
它更适合后续统一标准，原因主要是：

- 体积更小
- 更适合长期外接自研板
- 支持 `Virtual COM port`
- `SWD / SWV` 速率和长期使用体验更好

## 8. 采购优先级建议

### 8.1 最小启动采购
建议先买：

1. `NUCLEO-G071RB`
2. `X-NUCLEO-DRP1M1`
3. `0.96 寸 I2C OLED`
4. `INA219` 模块
5. `24LC02B / AT24C02` 模块

当前已有 `ST-LINK/V2` 时，可先不把 `STLINK-V3MINIE` 作为阻塞项。

### 8.2 第二批采购
建议补：

1. `STLINK-V3MINIE`
2. `NUCLEO-G0B1RE`
3. `STM32G071B-DISCO`

### 8.3 第三批采购
等主控、协议和 `I2C` 路径都稳定后，再考虑：

1. `BQ25792EVM`
2. `BQ76942EVM`
3. `EV2400`

## 9. 当前推荐组合

### 9.1 最适合现在的组合
- 当前已有：`STM32F103C8T6` 学习板 + `ST-LINK/V2`
- 新增采购：
  - `NUCLEO-G071RB`
  - `X-NUCLEO-DRP1M1`
  - `INA219`
  - `I2C OLED`
  - `24LC02B / AT24C02`

这套组合最符合当前项目目标：

- 既能继续当前学习节奏
- 又能把开发主线拉回 `STM32G0`
- 同时开始积累 `I2C / PD / power observation` 的可复用驱动和测试文档

### 9.2 不建议现在的组合
- 只继续买 `F103` 周边
- 一上来就重押昂贵 `BMS` 评估板
- 为了省预算直接跳过 `STM32G0` 参考开发板

这些做法都会让当前开源项目更像“零散实验”，而不是“逐步成型的平台”。
