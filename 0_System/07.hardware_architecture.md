## 1. 系统框图

```text
[Type-C 输入/输出口] ──VBUS──> [SoC（英集芯，型号外包待定）]
                                      │ I²C (SCL/SDA/INT)
                                      │
[MCU（Brain）] <───────────────────────┤
  STM32F103C8T6（当前 bring-up）       │
  STM32F030C8T6 类（量产候补，ADR-002） │
  │                                   │
  │ I²C（与 SoC 共线或独立总线，外包确认）
  │
  └──> [ST25DV NFC 标签（I²C 从设备）]
         写入：电池状态快照 + 关键日志摘要
         读出：手机 tap ISO15693 / NFC Forum Type 5

[USB-A 输出口]  ←── SoC 功率路径
[Type-C 输出口] ←── SoC 功率路径（PD/QC/UFCS 由 SoC 处理）

[NTC 温度传感器] ──ADC──> MCU
[按键] ──GPIO──> MCU
[LED（必须）/ 轻量 OLED（可选）] ←── MCU GPIO
```

三态标注：
- 双芯 MCU+SoC 框架（ADR-001）、NFC ST25DV 选型（ADR-003）：**已确认输入**。
- 电源路径细节、I²C 总线拓扑、引脚分配：**外包待填**。
- F030 量产候补（ADR-002）：**规划中，待样片验证**。

## 2. Power Tree（当前假设）

| 电源节点 | 来源 | 电压 / 电流 | 供给 | 三态 | 备注 |
|---------|------|-----------|------|------|------|
| VBUS | Type-C 输入 | 5–20 V（PD 协商） | SoC buck-boost 输入 | 当前假设 | 最高支持电压待 SoC 型号确认 |
| VSYS | SoC 升降压输出 | 待定（典型 5 V） | MCU LDO / 外设电源 | 外包待填 | 实际电压与容量由外包团队确认 |
| VDD_MCU | LDO / DCDC | 3.3 V | MCU VDD | 当前假设 | LDO 还是 DCDC 由外包方案决定 |
| VDD_NFC | LDO 或从 VSYS | 3.3 V | ST25DV VCC | 外包待填 | ST25DV 支持被动能量收割（RF 供电），需打样验证读距 |

> 打样前上述电压节点均为假设，不得用于阶段门验收。

## 3. 关键器件（当前已知 / 待补）

| 器件 | 型号 / 类型 | 角色 | 关联能力 | 三态 |
|------|-----------|------|---------|------|
| MCU | STM32F103C8T6 | Bring-up Brain；策略调度、日志、I²C 主控 | 全部 | 当前代码事实 |
| MCU（候补） | STM32F030C8T6 类（具体料号外包待定） | 量产 Brain（需样片验证，ADR-002） | F030_MIN_PRODUCT | 规划中 |
| 电源 SoC | 英集芯 IP53xx（型号外包待定） | buck-boost + PD/QC/UFCS + 硬件 OVP/OCP/OTP | Power / PD / 快充 | 外包待定 |
| NFC 标签 | ST25DV | 被动 NFC 数据读出窗口；I²C 写入 + RF tap 读出 | NFC | 已确认选型（ADR-003） |
| 温度传感器 | NTC（规格外包待定） | 环境/电芯温度采样 → MCU ADC | 温度保护 | 外包待填 |
| 电芯 | 待定 | 储能 | Battery | 外包待填 |
| LED | 待定（必须） | 最小状态指示 | UI | 外包待填 |
| OLED | 待定（可选） | 轻量数字页显示 | UI | 规划中（非 F030 最小版本必须） |

## 4. 接口划分（硬件 → 软件边界）

> GPIO / 引脚定义全部落 board 层（`board/board_stm32f103c8t6_usb_uart.h` 及未来的 `board_stm32f030_*.h`），不在 app 层硬编码。呼应"要接口不要源码"原则（ADR-001）。

| 接口 | 硬件侧 | 软件侧（board 层定义） | 三态 | 说明 |
|------|-------|---------------------|------|------|
| SoC INT | SoC INT 输出引脚 | PA<x>（EXTI，具体 pin 外包确认） | 外包待填 | 硬件安全中断高速通道；中断回调严禁 I²C/printf/delay |
| I²C（SoC） | SoC SCL/SDA | PB6/PB7 或待定 | 当前假设 | MCU 作主设备；总线地址待 SoC 型号确认 |
| I²C（NFC） | ST25DV SCL/SDA | 与 SoC 共线或独立，外包确认 | 外包待填 | ST25DV I²C 地址由 A0 引脚决定（0x53 / 0x57） |
| NTC / ADC | NTC 分压网络 | ADC 通道（pin 外包确认） | 外包待填 | 温度保护慢路径采样 |
| LED | LED + 限流电阻 | GPIO（pin 外包确认） | 外包待填 | — |
| 按键 | 机械按键 | GPIO（pin 外包确认） | 外包待填 | 建议上拉，低电平触发 |
| Type-C 连接器 | 输入/输出 CC 线 | 交由 SoC 处理；MCU 不直接接 CC | 已确认 | SoC 负责 PD 协商，MCU 通过 soc_api 读取协商结果 |

## 5. 待确认项（进阶段门前必须闭合）

以下字段为"外包待填"，在进入阶段门之前，卡2 需提供确认值，卡1 owner review：

1. **SoC 具体型号** → 对应 I²C 地址、寄存器 API 文档、INT 极性、最大电压范围
2. **VSYS 电压值** → 决定 MCU LDO 选型与 VDD_NFC 供电方案
3. **I²C 总线拓扑** → SoC 与 ST25DV 是否共用同一物理总线（影响地址冲突排查）
4. **SoC INT 引脚号** → board 层需更新 `BOARD_SOC_INT_PIN`
5. **NTC 规格** → ADC 分压参数（上拉电阻值、NTC 曲线）
6. **F030 候补具体料号** → 用于阶段门 Phase 3（ADR-002）
7. **ST25DV 天线开窗方案** → 金属机身失谐风险评估（ADR-003 遗留，打样验证）
