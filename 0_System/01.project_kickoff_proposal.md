## 一、项目背景与核心思路

### 1.1 战略定位

- 目标导向：以“占领日本市场心智”为核心，而非单纯追求技术参数。通过“**安全 + 数据可展示可追踪**”实现产品差异化，区别于传统“黑盒”充电宝。
- 角色定位：作为**面向日本中端市场的产品定义与系统整合方**，利用成熟电芯与专用电源 SoC（英集芯为当前合作主线）的能力，核心竞争力在于“集成能力”与“数据可读可追踪”的落地。

### 1.2 核心差异化策略

- 数据可读可追踪：把传统“盲充”转变为状态可读的智能电源系统。MCU 记录电池状态与关键日志（电量、充放电次数、过温/故障事件），用户通过 **NFC(ST25DV) tap 读取**当前状态，离开后再次 tap 取更新。数据用途为**安全合规 + 监控（记录）**；实时/隔空/远程监控/OTA 不在需求内。此处“可视化”指**数据可读**，非设备端屏幕。
- 技术路径：技术是手段而非目的。利用 SoC 的底层能力，通过 MCU 应用层开发，打造符合日本市场实际需求（高安全、高合规、精细化）的产品。

## 二、技术关键路径

### 2.1 IC 选型与底层架构

- **SoC 合作主线已统一到英集芯（Injoinic）**，代表芯片如 IP53xx 系列（单芯锂电、高集成度、Type-C 快充）。
- 软件保留 **SAL（SoC 抽象层）**多厂商边界，但当前只实现 1 个 vendor adapter，不做多 SoC 并行适配；智融（iSmartware，SW 系列）等可作为未来备选经 SAL 接入。
- 功率定位：**中端便携**，目标快充协议 **PD3.0 / PD3.1 SPR（~100W）**；PD3.1 EPR（>100W，28/36/48V）原则不在中端便携 / ≤100Wh 范围。

### 2.2 MCU 应用层开发（核心战场）

SoC 负责“体力活”（电压转换、协议握手、快保护），MCU 负责“脑力活”：

- 数据读取与计算：读取底层数据，计算 SOC（电量）、SOH（电芯健康度）、剩余可用时长。
- 记录与读出：**掉电持久化关键日志**；经 I²C 把状态快照 + 日志摘要写入 **NFC(ST25DV)**，供手机 / Web NFC tap 读出（可免 App）。
- 智能策略：按温度与环境动态降额；多设备充电时的功率智能分配。
- 安全慢路径：故障锁存、提示、恢复控制；不替代硬件快保护。

> 设备端液晶屏、App 常连、隔空 OTA 不在首发范围；LED 仅作最小指示。

## 三、供应链合作策略

### 3.1 SDK 合作模式：要“接口”不要“源码”

针对英集芯等厂商 SDK 仅向核心方案商开放的现状，确立合理的谈判边界：

- 拒绝黑盒：不接受完全封闭的“傻瓜式”芯片，无法满足日本市场的定制化需求。
- 拒绝源码：不要求开放底层驱动源代码（涉及 IP 泄露和安全隐患）。
- 核心诉求：要求开放标准 API 接口文档和库文件，比如：
  - 数据读取接口：`Get_Voltage()`、`Get_Temperature()`。
  - 参数配置接口：`Set_OCP_Threshold()`（过流保护阈值）。
  - 事件回调机制：快充握手成功通知、异常中断通知。

### 3.2 双赢合作基石

- 市场准入：利用我方在日本市场的 PSE 认证经验和渠道资源，帮助芯片厂降低进入门槛。
- 差异化互补：我方负责上层应用与算法，芯片厂负责底层稳定性，共同打造“英集芯 Inside”的高端差异化产品。
- 利益绑定：通过承诺规模化采购或签订独家代理协议，换取 SDK 接口权限与技术支持。

## 四、合规与风险控制

### 4.1 日本市场合规落地

- PSE 认证：必须通过日本电气用品安全法认证，重点测试电池安全与电磁兼容。
- 航空法规：严格控制电池容量（如 ≤100Wh），优化尺寸重量，满足便携出行需求。
- 无线认证（技適）：**NFC(ST25DV) 为被动标签，充电宝本身非主动发射机 → 基本免主动发射技適**（13.56MHz 仍受电波法约束，量产前须与认证机构确认口径）；若未来加 BLE 则必须过技適或用预认证模组。
- 形态：**磁吸为可选形态**（非必须适配 Qi2）；无线充电为首发外 P2 扩展项。

### 4.2 技术与市场风险应对

- 热设计风险：中端便携（目标 ~100W 内）小体积温控，采用石墨烯散热片、金属外壳导热。
- **NFC 天线风险**：13.56MHz 天线在金属外壳 / 电芯附近会失谐，需铁氧体屏蔽 + 天线开窗 / 避让，读距必须打样实测。
- **SoC 接口依赖**：英集芯真实接口能否满足状态读取 / 参数配置 / 事件模型，仍待验证。
- **市场假设**：日本中端客户是否愿为“数据可读可追踪”形成可感知溢价，是核心假设，需渠道 / 付费行为证据。
- 电芯一致性：优化 BMS 均衡算法，确保多节电芯并联 / 串联时的一致性，延长产品寿命。
