# Plan-Week-0325 item-02

## 1. 目标
确认首批关键硬件的采购状态、预计到货时间和联调顺序，让本周计划能按真实物流节奏执行，而不是继续建立在固定 3 天假设上。

## 2. 与主干目标的关系
当前路线已经拆成“F103 学习起步 + G0 长期参考 + 扩展板后续联调”三层。如果到货节奏不清楚，本周就无法合理安排“软件前置”和“实板联调启动”的边界。

## 3. 前置条件
- 首批推荐板卡已确定：
  - `STLINK-V3MINIE`
  - `NUCLEO-G071RB`
  - `X-NUCLEO-DRP1M1`
  - `BQ76942EVM`
- 已确认个人开发主路径为 `STM32CubeIDE for Visual Studio Code`

## 4. 实施步骤
1. 列出本周联调依赖的最小板卡组合，并按优先级排序：
   - 第一优先：`STLINK-V3MINIE`
   - 第二优先：当前 `野火小智 STM32F103C8T6 核心板（双 USB 款）`
   - 第三优先：`X-NUCLEO-DRP1M1`
   - 第四优先：`BQ76942EVM`
   - 第五优先：`NUCLEO-G071RB`
2. 记录每块板的实际状态：
   - 已下单 / 未下单
   - 已发货 / 未发货
   - 预计到货日期
   - 到货后首个验证目标
3. 明确本周联调门槛：
   - 若 `ST-LINK` 到位，本周必须先完成 F103 下载与调试链路确认
   - 若 `X-NUCLEO-DRP1M1` 到货，且 F103 日志链路已稳定，本周启动输入链路实板联调
   - 若 `BQ76942EVM` 同步到货，本周增加“上电与通信探测”
   - 若 `BQ76942EVM` 未到货，本周只保留 BMS 接口准备，不强行做实板闭环
4. 建立一张简单的到货跟踪表，字段建议如下：
   - 板卡名称
   - 当前状态
   - 预计到货时间
   - 对应联调动作
   - 风险备注
5. 每日收工前更新一次状态，确保 Day 4 以后执行内容跟着实物节奏调整。

## 5. 完成判定
- 能清楚知道本周哪些板能到、哪些板不能到
- 本周联调动作与板卡到货状态一一对应
- 不再使用固定“3 天后到货”作为唯一假设

## 6. 输出物
- 板卡到货跟踪表
- 本周联调优先级列表
- 到货触发条件说明

## 7. 注意事项
- 到货延迟时，不要压缩软件准备时间来追赶进度。
- 未到货的板卡只能影响“联调开始时间”，不应影响工程结构和日志体系的前置建设。
