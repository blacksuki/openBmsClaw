# hal (MCU 外设包装与基础硬件层)

本目录承接纯净的单片机内部基础外设（GPIO / I2C / UART / ADC 等）操作封装。在最新的“脑体协同”双核架构下，本目录严格遵守以下职能边界：

## 1. 核心定位与职责边界
- **仅做透传与保护**：`hal/` 不包含任何产品级功率计算、协议解析或专用 SoC（英集芯/智融）的业务逻辑。它仅向上层提供稳定的总线收发与管脚操控能力。
- **强制防死锁纪律**：所有底层外设驱动（尤其是 `hal_i2c`）必须具备严格的硬件或软件超时检测机制（Timeout），杜绝因从机响应延迟或死锁引起的死循环挂死。

## 2. 目录组织架构
```text
hal/
├── README.md       本说明文档
├── i2c/            标准 I2C 外设封装 (带超时保护与连续 9 时钟总线自愈机制)
├── uart/           标准 UART 外设封装
└── adc/            标准 ADC 外设封装 (用于基础电压与板载 NTC 温度监测)
```

> 注：早期废弃的 `bms/`、`power/`、`protocol/` 等散件思维辅助目录不再使用，所有外部业务通信均收敛至 `drivers/` (SoC SAL) 统一转译。
