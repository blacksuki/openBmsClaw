# 诊断脚本 (recognition 式,一次一问,按答案分叉,封顶约 7 问)

Q1(定位用): 你接触日本移动电源/储能这件事,更接近哪种处境?
  - 要采购/选型芯片或电芯       → 侧重节点 [4, 5]
  - 要卖给客户 / 做市场         → 侧重节点 [1, 2, 6]
  - 要做产品 / 研发            → 侧重节点 [2, 3, 7]
  - 要投资 / 评估这门生意       → 侧重节点 [1, 5, 6]
  - 只想看懂全貌              → 全节点浅讲 [1→7]

Q2(场景化探测): 假设下周你要向一个日本客户解释"我们为什么不是普通充电宝",下面哪个你"完全没概念"?
  - 数字化可视 vs 黑盒到底差在哪        → 侧重节点 [1, 2]
  - 它内部"脑体双芯"怎么分工           → 侧重节点 [3]
  - 进日本要过哪些合规关               → 侧重节点 [6]

Q3(按需追问): 你最担心哪一块出岔子?
  - 选错芯片 / 跟供应商谈不拢          → 侧重节点 [4, 5]
  - 产品做出来过不了认证 / 出安全问题    → 侧重节点 [6, 7]
  - (不担心 / 说不上来)              → 跳过,按 Q1/Q2 结果走
