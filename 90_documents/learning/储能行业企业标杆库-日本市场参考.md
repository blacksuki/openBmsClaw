# 储能行业企业标杆库-日本市场参考

## 1. 文档用途

这份文档用于服务 `openBattery` 后续进入储能行业的前期研究，目标不是给出一次性结论，而是形成一个可持续补充的 **企业标杆库模板 + 日本市场认证参考框架**。

适用场景：

1. 未来筛选潜在合作伙伴
2. 规划自有产品认证路径
3. 识别行业主流技术路线和交付能力
4. 为日本市场进入策略准备资料清单

当前判断前提：

1. 我们的企业是 **日本注册公司**，对外品牌包装按 **日本本土品牌** 运作。
2. 我们不是单纯按“中国企业出海日本”的路径做市场进入，而是按“**日本主体 + 国内供应链制造/采购能力**”来组织业务。
3. 因此，认证策略的核心不是“怎样把中国证书翻译给日本客户看”，而是：
   - 怎样让日本法人名义下的产品和系统在日本本地合规
   - 怎样把国内成熟供应链能力转化成日本市场可接受的产品与资料包
   - 怎样在品牌是日本本土的前提下，把关键零部件、系统、安全、网络安全责任链说明白

当前纳入范围：

- 已完成详细案例：`赣锋锂电 / 赣锋储能`
- 本轮补充标杆：`Sungrow`、`Huawei Digital Power`、`CATL`、`HiTHIUM`

说明：

- 本文优先关注 **储能系统产品化能力、认证能力、日本市场进入准备度**。
- 不把“公开宣传”直接当作“已落地事实”，尽量区分：
  - 已有官方/高可信公开材料支撑
  - 已见公开动作但仍需补证
  - 仅适合作为观察项

## 3. 日本市场认证策略前提

对当前业务背景，先把一个常见误区说清楚：

- 如果企业主体在日本、品牌也按日本本土品牌运作，**并不等于**认证压力会下降。
- 相反，责任会更集中地落在日本法人、品牌方、进口/销售主体、施工交付主体身上。
- 国内供应链可以提供成熟技术、制造和成本优势，但不能替代：
  - 日本市场本地认证
  - 日本市场并网要求
  - 日本市场网络安全要求
  - 日本市场售后与责任主体要求

因此更合理的认证策略是：

1. **产品安全和系统安全** 采用国际主流标准打底
2. **日本本地准入** 采用 `JET + JIS + 系统连系 + JC-STAR` 补齐
3. **运输与交付资料** 采用国际通用资料包保证供应链和项目执行
4. **日本法人责任链** 明确谁是品牌方、进口方、销售方、安装方、售后方

## 4. 认证必须项目

这里按你当前背景，把认证拆成三层：

- `必须项目`：不建议后补，应该在产品定义阶段就写进输入条件
- `强烈建议项目`：不是所有场景都强制，但大概率会影响销售、项目签约或客户信任
- `项目型补充项`：视场景、渠道、市场和客户要求补齐

### 4.1 必须项目总表

| 项目 | 适用对象 | 目的 | 重要度 | 备注 |
|---|---|---|---|---|
| `IEC 62619` | 电芯 / 模组 / 电池包 / 电池系统 | 证明工业用锂二次电池及系统基础安全性 | `必须` | 是储能电池系统的国际基础安全语言 |
| `IEC 62477-1` | PCS / 电力电子 / 储能系统 | 证明电力电子变换设备安全 | `必须` | 尤其适合带 PCS 的系统方案 |
| `IEC 61000-6-2 / -6-4` | 系统 / PCS / 控制设备 | 证明抗扰度与发射 EMC 能力 | `必须` | 日本客户和项目资料里很常见 |
| `UN38.3` | 电池及含电池系统出货 | 满足运输前的锂电运输测试要求 | `必须` | 没有它，很多出货链路直接卡死 |
| `MSDS / SDS` | 电池及整机出货 | 支持运输、仓储、项目交付与客户审查 | `必须` | 日本本地品牌也不能省 |
| 海运运输鉴定 / 危险品分类 | 大储柜 / 集装箱 / 大型系统 | 支持海运、报关、仓储、项目交付 | `必须` | 尤其是用国内供应链供货到日本 |
| `JET 系统连系保护装置等认证` | PCS / 并网相关装置 | 满足日本并网与系统连系要求 | `必须` | 面向日本市场时不能后补 |
| `JIS C 4412 / S-JET` | 低压蓄电系统 | 满足日本低压储能系统产品认证 | `必须（低压系统）` | 户储和小型工商业尤其重要 |
| `JIS C 4441` | 大型定置储能系统 | 满足日本大型储能系统传播/安全要求表达 | `必须（大储）` | 面向工商业柜/大储应前置考虑 |
| `JC-STAR ★1` | 具 IP 通信能力的控制系统 | 满足日本网络安全标签及并网要件趋势 | `必须（日本并网型通信系统）` | 2027 年起对太陽光/蓄電池并网影响实质化 |

### 4.2 强烈建议项目

| 项目 | 适用对象 | 目的 | 重要度 | 备注 |
|---|---|---|---|---|
| `UL 1973` | 储能电池系统 | 为北美客户、国际客户和部分日本客户提供更强背书 | `强烈建议` | 对“国际化成熟度”帮助大 |
| `UL 9540 / UL 9540A` | 系统 / 柜 / 集装箱 | 证明系统级安全和热失控蔓延能力 | `强烈建议` | 日本客户未必强制，但对大客户很有说服力 |
| `IEC 62933-5-2` | 大型储能系统 | 提供定置储能系统国际安全表达 | `强烈建议` | 与日本 `JIS C 4441` 有很强关联 |
| `ISO 9001 / 14001 / 45001` | 供应链 / 工厂 / 品牌方 | 证明质量、环境、职业健康管理能力 | `强烈建议` | 对供应链审厂和品牌可信度有帮助 |

### 4.3 项目型补充项

| 项目 | 适用对象 | 目的 | 重要度 | 备注 |
|---|---|---|---|---|
| `NFPA 855` 等北美消防要求 | 北美项目或国际比选项目 | 适应北美消防与储能部署要求 | `项目型` | 非日本本地必须 |
| 客户定制网络安全审查 | EMS / 云平台 / 远程运维 | 满足项目级 IT / OT 安全要求 | `项目型` | 日本大客户越来越重视 |
| 日本本地施工与电气配套资料 | 系统交付 | 满足设计审查、施工与售后 | `项目型` | 常不是单一证书，而是资料包 |

### 4.4 每个“必须项目”的目的和重要度说明

#### `IEC 62619`

- 目的：证明储能电池相关部件和系统具备基础安全性。
- 为什么重要：它几乎是进入储能系统对话的底线。如果连这项都没有，后面很多系统认证、客户审查和项目沟通都难展开。
- 对你们的意义：即使品牌是日本本土，只要核心电池系统来自国内供应链，这项仍然是基础起点。

#### `IEC 62477-1`

- 目的：证明储能系统中的电力电子设备安全，尤其适合 PCS 和带功率变换功能的系统。
- 为什么重要：日本市场不会因为你是日本品牌就降低 PCS/功率变换系统的安全要求。
- 对你们的意义：如果未来产品形态不是单纯电池包，而是完整系统，这项应该前置。

#### `IEC 61000-6-2 / -6-4`

- 目的：证明设备的电磁兼容能力，包括抗干扰和发射控制。
- 为什么重要：它直接关系到项目现场能否稳定运行，也关系到客户对系统成熟度的判断。
- 对你们的意义：这项是“工程产品像不像工程产品”的基础信号。

#### `UN38.3 + MSDS + 运输鉴定`

- 目的：让带锂电池的设备能够合规运输、仓储和交付。
- 为什么重要：这是供应链和项目执行的生命线，不是锦上添花。
- 对你们的意义：即使日本法人卖货，只要制造、采购、集成主要依托国内供应链，这几项就是供应链闭环的必要条件。

#### `JET 系统连系保护装置等认证`

- 目的：满足日本并网和系统连系的本地要求。
- 为什么重要：不解决这个问题，很多并网型产品根本谈不上真正落地。
- 对你们的意义：因为你们走的是日本本土品牌路线，这项比“国际品牌出海宣传材料”更重要，属于必须正面面对的本地准入问题。

#### `JIS C 4412 / S-JET`

- 目的：满足日本低压蓄电系统的本地认证要求。
- 为什么重要：它对应的是日本市场真正会看、会用、会卡你的本地认证语言。
- 对你们的意义：如果从户储或小型工商业切入，这项应被视为必须项。

#### `JIS C 4441`

- 目的：用于大型定置储能系统的安全与传播性能表达。
- 为什么重要：对工商业柜和大储系统，日本不会只看欧美标准。
- 对你们的意义：如果后面走到工商储柜、集装箱或更大系统，这项不能等到客户要求时再补。

#### `JC-STAR ★1`

- 目的：证明具 IP 通信能力的控制系统满足日本 IoT 产品网络安全基线要求。
- 为什么重要：它已经不只是“可有可无的标签”，而是正在进入日本储能并网的实质性准入条件。
- 对你们的意义：你们虽然是日本公司，但如果系统里有 `PCS / EMS / BMS / Gateway / Remote monitoring` 等通信与控制链路，仍然绕不过去。

## 5. 日本市场认证参考

结论先说：如果未来要做日本储能市场，不能只盯着 `IEC / UL` 这类国际标准，还要把 **JET + JIS + 系统连系** 放进最早期设计输入。

### 5.1 电芯 / 模组 / 电池系统级

优先参考：

1. `JIS C 8715-2`
   - JET 已将其作为工业用锂二次电池相关部件认证的重要依据。
   - 适用于单电池、电池模组、电池包、电池系统等部件级认证参考。

2. `Propagation test / 类烧试验`
   - JET 明确把 `JIS C 8715-2` 的设计评价和类烧试验能力纳入了认证表达。
   - 对未来强调“热失控不蔓延”的产品很关键。

### 5.2 低压储能系统

优先参考：

1. `S-JET 认证（蓄电系统）`
2. `JIS C 4412`

意义：

- JET 将 `JIS C 4412` 用于低压蓄电系统产品认证。
- 它不仅看系统安全，还包含电波障害等追加要求。
- 同时，JET 明确说明该认证包含与系統连系保护装置等认证相同的部分要求，便于后续拿并网相关认证。

### 5.3 定置用大型储能系统

优先参考：

1. `JIS C 4441`
2. `IEC 62933-5-2`

意义：

- JET 已推出基于 `JIS C 4441` 的大型定置储能系统认证。
- JET 明确说明 `JIS C 4441` 是日本提出的、与 `IEC 62933-5-2` 国际整合的标准。
- 这意味着如果未来做大储/工商业柜/集装箱产品，日本市场不能只拿欧美认证，还要考虑日本本地化安全表达。

### 5.4 PCS / 并网 / 系统连系

优先参考：

1. `JET 系統連系保護装置等认证`

意义：

- 日本市场对于 PCS、并网保护、系统连系的要求不是“有逆变器就能卖”。
- JET 的系统连系保护装置等认证，直接服务于分布式发电和储能相关并网装置的性能与安全确认。
- 对低压系统尤其重要，未来如果做户储或小型工商业储能，这一步不能后补。

### 5.5 JC-STAR 分析结论

这里单独说明，因为它对你们路径很关键。

#### `JC-STAR` 的本质

- `JC-STAR` 是日本的 IoT 产品网络安全标签制度。
- 它本身不是只给储能行业准备的认证，而是更广泛的网络安全合规制度。
- 但对储能行业来说，只要产品里存在具 IP 通信能力的控制系统，它就会变成实质性要求。

#### 对储能行业的真实影响

已经能确认的结论：

1. 对太陽光和蓄電池项目，日本已经明确了把 `JC-STAR ★1` 嵌入系統連系技术要件的方向。
2. 高压/特高压适用起点为 `2027年4月`。
3. 低压 `50kW` 未满适用起点为 `2027年10月`。
4. 实务上更关键的时间点是 **接入合同申请日**，不是完工日。
5. 具通信功能的控制系统，如 `PCS / EMS` 等，是重点对象；其他具控制能力或直连主要设备的通信设备，也可能落入范围。

#### 应如何理解它的“重要度”

更准确的说法不是：

- “JC-STAR 已经是所有储能认证的唯一门票”

而是：

- “JC-STAR 正在从自愿网络安全标签，快速转变为日本并网型储能系统中的实质性准入要求”

对你们这种路径尤其重要，因为：

1. 你们不是以中国品牌身份先做海外试销，而是以日本本土品牌进入市场。
2. 日本客户和监管方会默认你们应该从一开始就满足日本本地网络安全要求。
3. 一旦系统中存在远程监控、云平台、PCS 通信、EMS 调度、网关、聚合控制等能力，`JC-STAR` 就不能被当作后补项。

#### 当前不能写死的地方

以下表述建议保留谨慎口径：

1. 不能简单写成“所有日本储能补贴都以 `JC-STAR ★1` 为前提”。
2. 不能简单写成“所有容量市场/长期脱碳拍卖已统一把 `JC-STAR` 作为唯一准入门槛”。
3. 更稳妥的写法是：
   - `JC-STAR` 已明显进入并网和部分项目/补贴/市场机制的合规审查语境，未来影响会继续扩大，需逐项目核对公募要领和募集要纲。

#### 对 openBattery 的结论

对你们当前的日本本土品牌路线，`JC-STAR` 应被视为：

- `必须前置分析项`
- `并网型产品的高优先级认证/合规项`
- `系统架构阶段就要考虑的网络安全输入条件`

### 5.6 出海和交付资料基线

即使目标市场是日本，以下资料也仍然是主流基础包：

1. `IEC 62619`
2. `IEC 62477-1`
3. `IEC 61000-6-2/-4`
4. `UL 1973`
5. `UL 9540 / UL 9540A`
6. `UN38.3`
7. `MSDS`
8. 海运运输鉴定 / 危险品分类 / 项目交付规格书

建议理解方式：

- `IEC / UL`：全球市场和大客户交流的通用语言
- `JET / JIS / 系统连系`：进入日本市场时必须前置考虑的落地语言

## 5.2 Sungrow

### 基本判断

- 类型：`PCS + ESS 系统型企业`
- 适合作为：`日本市场进入动作`、`JET 路径参考`、`PCS/系统一体化表达` 的重点对标对象

### 公开强项

1. 不只是做逆变器，也长期做储能系统与 PCS。
2. 日本本地动作明确，既参加日本展会，也有日本合作伙伴与销售协议动作。
3. 在日本市场公开展示了 `JET` 认证成果，这一点对我们做日本参考价值很高。

### 日本市场参考价值

重点看三件事：

1. 日本本地站点与产品入口已经成熟
2. 2025 年在日本 Smart Energy Week 展示储能与 PCS 产品
3. 2025 年公开披露住宅储能系统获得 `JET` 认证，并与日本本地伙伴推进储能系统采购/销售合作

### 建议重点收集的资料

1. `JET` 证书样式与适用产品范围
2. 日本市场销售资料中的安装、并网、售后说明
3. 户储与工商业系统的日文规格书
4. `PCS + 电池 + EMS` 的系统边界定义方式

### 标杆判断

- 标杆等级：`A`
- 原因：它兼具全球储能系统能力与日本本地市场动作，对“如何真的进入日本”比单纯全球大厂更有参考意义。

### 对 openBattery 的借鉴

1. 日本市场不是只翻译资料，而是要有 **本地认证 + 本地合作 + 本地产品命名/规格表达**。
2. 如果未来要进入储能，日本线产品最好从一开始就考虑 `JET` 和系统连系要求。

## 5.3 Huawei Digital Power

### 基本判断

- 类型：`数字能源平台型企业`
- 适合作为：`户储/工商业日本市场参考`、`光储一体资料表达`、`证书管理方式` 的重点对标对象

### 公开强项

### 日本市场参考价值

它的优势不在“大储出货榜故事”，而在：

1. 日本向资料组织得更像“可落地销售与交付资料”
2. 证书公开度较高，适合研究别人怎么对外展示合规能力
3. 光储和户储方案的日本本地适配值得参考

### 建议重点收集的资料

## 8. 对 openBattery 的阶段性建议

如果未来本企业计划进入储能行业，建议按下面顺序准备，而不是一开始就冲产品线广度。

### 8.1 第一阶段：先补齐资料语言

最先要补的不是“更多型号”，而是统一资料结构：

1. 产品规格书
2. 安规/EMC/运输资料清单
3. 系统结构图
4. 典型应用场景说明
5. 面向客户/渠道的认证解释页

### 8.2 第二阶段：先定义日本线认证策略

建议从立项阶段就把下列问题写进输入条件：

1. 面向日本的是户储、工商业还是更大系统
2. 是否需要 `JET` 路径
3. 是否需要 `JIS C 4412 / JIS C 4441`
4. PCS 是否涉及并网与系统连系保护认证
5. 系统是否涉及 `JC-STAR` 约束对象
6. 出货形态是否涉及 `UN38.3 / MSDS / 海运鉴定`

### 8.3 第三阶段：先做一条最小可落地样板线

比起一开始做很多型号，更建议先定义一条样板线：

1. 一个明确场景
2. 一套明确资料包
3. 一条明确认证路径
4. 一个可展示的日本市场故事

例如可以理解为：

- `样板产品`
- `样板认证包`
- `样板渠道资料`
- `样板日文资料`
